# Mobile Fix

Apply mobile-efficiency fixes to source files based on a `/mobile-audit` report. Reads the audit, applies the changes, reports a verification checklist.

Will NOT generate its own findings. If you don't have an audit yet, run `/mobile-audit` first.

## When to use

After `/mobile-audit` has produced a report. Either:

- Pass an audit file: `/mobile-fix .claude/audits/mobile-2026-04-26-habits.md` — apply that specific report.
- Pass nothing: pick the most recent file in `.claude/audits/` (sort by mtime, descending).
- Pass a target name like `habits`: load `.claude/audits/mobile-*-habits.md` (most recent match).

## Inputs (`$ARGUMENTS`)

- A path to an audit file
- A target name (matched against audit slugs)
- (empty) — most recent audit

## Steps

1. **Locate the audit file** per the rules above. If the file doesn't exist, output `No audit file found. Run /mobile-audit first.` and stop.

2. **Parse the audit.** Extract findings grouped by source file. For each finding, capture: severity, title, line range, current snippet, fix snippet, optional notes.

3. **Filter:** by default apply only **High** and **Medium** findings. Skip **Low** unless `$ARGUMENTS` includes the literal token `--all`.

4. **Confirm before mass edits.** If 5+ files are affected or 10+ total findings, output a brief summary first:
   ```
   About to apply 14 findings across 4 files (10 High, 4 Medium). Skipping 6 Low.
   Files: habits/page.tsx (5), media/page.tsx (4), shopping/page.tsx (3), reading/page.tsx (2)
   Reply "go" to apply, or "stop".
   ```
   Wait for confirmation. For smaller scopes (1–2 files, ≤5 findings), just proceed.

5. **For each source file with findings:**
   - Read the file once.
   - Apply each finding in order using `Edit` against the `Current:` → `Fix:` pair.
   - If a `Current:` snippet doesn't match the file (file changed since audit ran), skip that finding and report `Stale: <file> finding N — current snippet not found, skipping.`
   - If a finding's `Notes:` say it needs manual review, skip it and report `Manual: <file> finding N — <notes>.`
   - If the file uses unusual structure (custom layout component, complex conditional rendering you can't reason about confidently), skip it and report `Manual: <file> — needs manual review.`

6. **Report results to chat:**
   - One line per fix applied: `Fixed habits/page.tsx finding 1 — wide row now stacks`
   - One line per skip with reason
   - Final summary: `Applied X / Y findings across Z files.`

7. **Output a verification checklist** at the end. Tailor it to what was actually changed:
   ```
   Verification:
   - Open <page> in a 375px-wide browser
   - Check: filter chips scroll horizontally (don't wrap)
   - Check: habit row stacks vertically (name on top, strip below)
   - Check: no horizontal page overflow
   - Check: sticky header stays readable
   ```

8. **Do NOT stage or commit.** Leave changes for the user to review and run `/commit` separately.

## Rules

- **Audit-driven only.** Never invent fixes. If a finding isn't in the audit, don't apply it.
- **Edit, never Write** for source files — surgical changes only. Whole-file rewrites lose unrelated work.
- **Stop on first ambiguity.** If two findings overlap on the same lines, apply only the first; report the second as `Conflict: skipped`.
- **Preserve the user's voice in the file.** Match indentation, quote style, and class ordering of the surrounding code.
- **Never delete content** as part of a "fix" unless the audit explicitly says to remove an element.
- **Stale audits expire.** If the audit file is older than 7 days OR if any source file's mtime is newer than the audit, warn before applying:
  ```
  Audit is N days old (or file X has been modified since the audit ran). Findings may be stale. Reply "go" to proceed anyway, or "stop" and re-run /mobile-audit.
  ```
- **No build, no tests** — same as `/commit`, the dev server will surface issues.

## Interaction with /mobile-audit

The two skills are decoupled but share the audit file format. If `/mobile-audit`'s report format changes, this skill's parser must change in lockstep. Currently expected:

```
## <file path>

### Finding N — <Severity> — <Title>
**Lines:** A–B
**Issue:** ...
**Current:**
```
<short code snippet>
```
**Fix:**
```
<short code snippet>
```
**Notes:** (optional)
```

If the audit file's format doesn't match, output `Audit file format unrecognized. Was it generated by /mobile-audit?` and stop.
